# VoxCanvas

VoxCanvas 是一款面向“AI 语音绘图工具”赛题开发的纯语音控制绘图应用。用户通过浏览器麦克风说出“智能绘图，……”形式的命令，即可完成画布控制、基础图形创建、图形编辑、复杂场景生成、场景对象增量添加、历史操作和作品导出。

项目的核心目标是让用户尽量不依赖鼠标和键盘，用自然语言完成绘图创作。系统会优先使用本地规则处理低风险、低延迟命令；当命令涉及复杂语义、场景生成、语音纠偏或多步编辑时，再调用 AI Command Planner 将自然语言转换为标准绘图命令。

展示视频地址：[https://www.bilibili.com/video/BV1PaJK65Esh/](https://www.bilibili.com/video/BV1PaJK65Esh/?vd_source=8315b74c6b71c50e7b34a49ae46937a1)

## 第三方库与框架

本项目使用了以下主要第三方库和框架：

- React：构建前端交互界面。
- React DOM：渲染 React 应用。
- Vite：前端开发服务器与构建工具。
- TypeScript：提供静态类型检查。
- Express：提供本地 Node.js API 服务，用于转发 AI Planner 请求。
- Zod：校验后端接收的请求结构。
- dotenv：读取本地 `.env` 环境变量。
- concurrently：同时启动前端 Vite 服务和后端 Express 服务。
- Vitest：运行单元测试。
- ESLint、typescript-eslint、eslint-plugin-react-hooks、eslint-plugin-react-refresh：代码规范检查。

此外，语音识别能力使用浏览器 Web Speech API。AI 语义规划默认接入 DeepSeek Chat，通过本地后端读取 `DEEPSEEK_API_KEY`，不会把密钥暴露给浏览器端。

## 系统功能

### 1. 纯语音唤醒与命令输入

用户无需点击按钮，页面加载并获得麦克风权限后会持续监听语音。系统只执行唤醒词之后的内容，例如：

```text
智能绘图，画一个红色圆形在左上角
```

系统支持“智能绘图”“智能画图”“小画助手”等唤醒词及部分相近发音，用于降低语音识别不准确带来的影响。

### 2. 基础画布控制

支持通过语音执行基础画布操作：

- 清空画布
- 撤销
- 重做
- 调整画布尺寸
- 导出图片

导出支持 PNG、JPG、SVG。若用户只说“导出图片”但没有说明格式，系统会进入格式澄清流程，等待用户继续说“PNG”“JPG”或“SVG”。

### 3. 基础图形创建

支持创建基础可编辑图形：

- 圆形
- 矩形
- 三角形
- 线条
- 文本

基础图形可以带有颜色、尺寸和位置描述，例如：

```text
智能绘图，画一个红色圆形在左上角
智能绘图，插入很厉害三个字
```

### 4. 图形编辑

系统支持对已有对象进行编辑：

- 移动
- 相对移动
- 空间关系移动
- 改色
- 缩放
- 删除

对于 AI 生成的场景对象，系统会保留语义信息，例如“房子”“屋顶”“树”“树冠”“太阳”等。用户可以编辑整个对象，也可以编辑对象的某个部件：

```text
智能绘图，把房子往右移动一点
智能绘图，把房子的屋顶变成黑色
智能绘图，把树的树冠改成黄色
```

### 5. 复杂场景生成

用户可以直接描述一个完整场景，AI Planner 会将场景拆解为一组基础图形，并生成可编辑的 scene graph：

```text
智能绘图，画一个草地上的小房子场景，左边有一棵树，右上角有太阳
```

生成结果不是一张不可编辑的图片，而是由矩形、圆形、三角形、线条、文本等基础元素组成。每个元素会带有 `groupId`、`groupLabel`、`partLabel` 等语义信息，便于后续继续编辑。

### 6. 已有场景增量添加

在已有场景基础上，用户可以继续添加新对象，系统会尽量保持原场景不被重建：

```text
智能绘图，在太阳下面再加一朵云
智能绘图，在房子的右边再生成一棵树
```

新增对象会被分配独立的语义分组，避免多个相同对象混在一起，方便后续单独引用。

### 7. 多步命令拆解与顺序执行

系统支持用户在一句话中表达多个操作，并按顺序执行：

```text
智能绘图，把房子往右移动一点，屋顶变成黑色
智能绘图，把树缩小一点，然后把树冠改成黄色
```

AI Planner 会返回 `batch` 命令，系统再逐步执行其中的创建、移动、改色、缩放、对齐、排列等子命令。

### 8. 批量操作与对齐排列

支持对多个对象进行批量编辑：

```text
智能绘图，让两个圆左对齐
智能绘图，把所有圆形变成红色
智能绘图，把三个圆排成一行
```

系统会根据目标类型、数量、范围等信息匹配画布中的对象，并在目标不明确时提示用户进一步说明。

### 9. 歧义目标澄清

当系统发现多个对象都符合用户描述时，不会直接执行危险编辑，而是提示用户补充说明。例如多个绿色圆形同时存在时，用户说“把绿色圆形移动到上方”，系统会提示存在多个候选目标，等待用户继续说明具体对象。

### 10. 取消意图处理

系统支持识别取消类表达，避免误操作：

```text
智能绘图，画一个蓝色圆形，取消
智能绘图，算了
智能绘图，不要执行
```

如果系统判断用户表达的是取消意图，会终止本次命令，不修改画布。

### 11. 执行反馈与场景描述

界面会展示：

- 当前语音识别文本
- 本地解析结果
- AI Planner 结果
- 执行反馈
- 对象数量、撤销/重做数量、画布尺寸
- 当前场景的语义描述
- 画布对象列表

这些信息用于帮助用户确认系统理解了什么、执行了什么、影响了哪些对象。

## 功能实现方法

### 前端实现

前端使用 React + TypeScript + Vite 实现。主要模块包括：

- `VoiceInputPanel`：负责语音输入、唤醒词识别、命令处理流程和反馈展示。
- `useVoiceInput`：封装浏览器 Web Speech API，提供持续监听、自动重启、临时识别文本和最终识别文本。
- `DrawingCanvas`：使用 SVG 渲染画布和所有图形对象。
- `CanvasSceneSummary`：根据当前图形状态生成持续展示的场景语义描述。
- `CommandPreview`、`CommandFeedbackPanel`、`PlannerPreview`：展示命令解析、AI 规划和执行反馈。

### 命令解析流程

系统采用“本地规则 + AI Planner”的混合策略：

1. 本地 parser 先处理低风险命令，例如清空、撤销、重做、导出、简单基础图形创建。
2. 对于文本、复杂场景、语义对象编辑、批量编辑、画布尺寸调整、语音纠偏等命令，进入 AI Planner。
3. AI Planner 返回标准 JSON 命令，例如 `create`、`move`、`recolor`、`resize`、`scene`、`addSceneObject`、`batch`。
4. 前端对 AI 返回结果进行校验，避免执行不安全或不完整的命令。
5. 命令通过校验后进入 canvas operations，更新画布状态。

### 后端实现

后端使用 Express 实现本地 API：

- 读取 `.env` 中的 `DEEPSEEK_API_KEY`。
- 接收前端传来的用户语音文本、本地解析结果和当前画布上下文。
- 构造包含画布对象、语义分组、允许图形类型、允许颜色、坐标空间、few-shot 示例的 prompt。
- 调用 DeepSeek Chat。
- 将模型返回的 JSON 交给前端进行二次校验和执行。

后端只在本地使用 API Key，浏览器端不会直接接触密钥。

### 画布与场景结构

画布状态由 `CanvasState` 管理，包括：

- 画布宽高
- 图形对象列表
- 当前选中对象或语义分组
- undo/redo 历史

AI 场景生成会输出 scene graph，每个元素包含：

- 基础图形类型
- 颜色
- bbox 坐标
- 层级顺序
- 语义分组 ID
- 语义对象名称
- 部件名称

执行时系统会把这些元素转换为真实 SVG 图形，并保留语义元数据，方便后续通过语音继续编辑。

### 安全与容错策略

系统在执行前会进行多层检查：

- 不明确的危险编辑命令不会直接执行。
- 多个候选目标会触发澄清流程。
- 缺少导出格式时会触发格式澄清。
- 取消类表达会拦截当前命令。
- AI 返回的命令必须符合标准 schema 和当前画布上下文。
- 多步命令必须使用 `batch`，避免只执行其中一步造成误解。

## 本地运行方法

安装依赖：

```bash
npm install
```

复制环境变量文件：

```bash
cp .env.example .env
```

在 `.env` 中配置：

```bash
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_MODEL=deepseek-chat
```

启动前端和后端：

```bash
npm run dev
```

默认服务：

- 前端页面：`http://localhost:5173`
- 后端 API：`http://localhost:8787`

建议使用 Chrome 浏览器，并允许页面访问麦克风。
